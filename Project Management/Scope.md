# Project Scope

## Goal
Build a functional console-based Java application called Dimensional Dex within 7 days. The project must clearly demonstrate object-oriented programming, custom object design, and required data structures including ArrayList, HashMap, PriorityQueue, and Stack.

## In Scope
- Pokemon data model
- Pokedex manager logic
- Add, remove, search, display, and filter features
- Team builder
- Encounter system using PriorityQueue
- Evolution history using Stack
- Custom sorting algorithm
- Comparable implementation in Pokemon
- Sample Pokemon dataset
- Console menu for interaction

## Out of Scope
- Full battle system
- GUI with JavaFX or Swing
- Networking or database support
- Large-scale save system
- Advanced animation or visual effects
- Hundreds of Pokemon entries

## Seven-Day Schedule

### Day 1
- Create project structure and Java class files
- Build the Pokemon class with all required fields
- Decide default Comparable behavior

### Day 2
- Build DexManager
- Implement add, remove, search, and display
- Add sample Pokemon entries

### Day 3
- Implement filtering by type and dimension
- Test map-based lookup and list-based storage
- Clean up model and manager responsibilities

### Day 4
- Implement custom sorting algorithm
- Add sorting by level, attack, or ID
- Verify Comparable<Pokemon> behavior

### Day 5
- Build encounter system using PriorityQueue
- Set encounter priority rules
- Test encounter order with sample Pokemon

### Day 6
- Build evolution history using Stack
- Add undo evolution feature
- Implement team builder methods

### Day 7
- Connect everything through a console menu
- Test edge cases and fix bugs
- Prepare the project for submission or demo

## Success Criteria
- All required data structures are used correctly
- Code is organized into clear classes with separate responsibilities
- User can manage Pokemon, teams, encounters, and evolution history
- Project is stable enough to demo in a console run
